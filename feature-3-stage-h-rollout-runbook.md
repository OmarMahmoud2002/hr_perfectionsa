# مرحلة H - خطة الإطلاق التدريجي (Daily Performance)

تاريخ التنفيذ: 2026-03-29
الحالة: جاهز للتفعيل

## 1) خطة التفعيل الداخلي (Week 1 Pilot)

1. اليوم 1:
- تفعيل الميزة على جميع حسابات employee + admin/manager/hr/user.
- متابعة أولية لسجلات الإدخال والتقييم.

2. اليوم 2-3:
- مراجعة معدل التسجيل اليومي.
- مراجعة أخطاء الرفع (نوع الملف/الحجم/العدد).

3. اليوم 4-5:
- مراجعة سلوك الفلاتر وسرعة صفحة التقييم.
- جمع ملاحظات UX من موظفين فعليين.

4. اليوم 6-7:
- تطبيق تحسينات صغيرة حسب الملاحظات.
- إعلان جاهزية الإطلاق الكامل.

## 2) مؤشرات المتابعة اليومية

1. نسبة الموظفين الذين سجلوا الأداء اليومي.
2. عدد السجلات بدون مرفقات.
3. عدد التقييمات اليومية المنفذة.
4. نسبة أخطاء الرفع والتحقق (Validation errors).
5. زمن تحميل صفحة التقييم عند الفلاتر المختلفة.

## 3) Checkpoints قبل الإطلاق الكامل

1. التأكد أن كل مسارات daily-performance تعمل بصلاحيات صحيحة.
2. التأكد من سلامة تحميل المرفقات وفتحها.
3. التأكد من عمل تقييم النجوم والتعديل لنفس المقيم.
4. التأكد من ظهور تقييمات الموظف في صفحة الأداء اليومي.
5. التأكد من عدم وجود مشاكل في Blade cache / route cache.

## 4) خطة الرجوع السريع (Rollback)

1. إخفاء روابط الواجهة مؤقتا من القائمة الجانبية.
2. تعطيل المسارات الجديدة مؤقتا عبر تعليق route groups الخاصة بالميزة.
3. الإبقاء على البيانات في الجداول الجديدة دون حذف لحين حل المشكلة.
4. إعادة تفعيل الميزة بعد الإصلاح واختبار smoke.

## 5) أوامر التحقق السريع بعد النشر

```bash
php artisan route:list | grep daily-performance
php artisan view:cache
php artisan test --filter=DailyPerformanceFeatureTest
```

## 6) ملاحظات تشغيل

1. الميزة تعتمد على ربط user بالـ employee.
2. الميزة تعتمد على public storage للمرفقات.
3. الميزة متوافقة مع نظام الأدوار الحالي بدون تغيير في middleware الأساسي.
